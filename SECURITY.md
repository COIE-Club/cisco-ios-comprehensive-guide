# Security Policy

We take the security and accuracy of the **Cisco IOS Comprehensive Guide** very seriously. We appreciate the community's efforts in helping us maintain a high standard, and we ask that you responsibly disclose any critical findings.

## Supported Versions

Currently, the project is in active development. Only the latest commit on the `main` branch is officially supported for security and critical updates. 

| Version | Supported          |
| ------- | ------------------ |
| `main`  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

Please **do not** report security vulnerabilities or severe configuration flaws through public GitHub issues. 

Instead, please report them privately via email to: **coieclub@outlook.com**.

### What to include in your report:
To help us triage and resolve the issue quickly, please provide the following details:
* **Description:** A detailed explanation of the vulnerability or critical flaw and its potential impact.
* **Location:** The specific folder, file name, or line of text where the vulnerability exists.
* **Reproduction:** Steps to reproduce the issue (if applicable).
* **Mitigation:** Any proposed solutions, corrected Cisco IOS commands, or mitigations.

### Our Response Process:
1. **Acknowledgment:** We will acknowledge receipt of your report within a maximum of **3 days**.
2. **Triage:** We will investigate the issue and determine its validity and severity.
3. **Resolution:** If a fix is required, we will work on a patch and notify you when it is applied to the `main` branch.
4. **Recognition:** We may publicly acknowledge your contribution to our project (with your permission) once the fix is safely deployed.
